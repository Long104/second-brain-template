# note-3 — Plain PARA Starter Vault

A **minimal, tool-agnostic** personal knowledge management vault based on the [PARA method](https://fortelabs.com/blog/para/). No opinions, no design, no centric theme — just a clean skeleton you can make your own.

## What You Get

| Component       | What it does                                                |
| --------------- | ----------------------------------------------------------- |
| **inbox/**      | Dump everything here. Sort later.                           |
| **wiki/**       | PARA knowledge base — projects → areas → resources → archive |
| **dailyNote/**  | Daily journal entries. One file per day.                    |
| **todo/**       | Task tracking — pending, done, plans.                       |
| **templates/**  | Reusable templates with YAML frontmatter.                   |
| **AGENTS.md**   | Assistant (Jarvis) rules. Handles ingest, organization, QA. |

## Quick Start

1. **Clone or copy this folder** anywhere you want your vault.
2. **Open with your editor** — VS Code, Obsidian, iA Writer, whatever.
3. **Tell Jarvis what to do** — feed it through an AI interface that reads `AGENTS.md`.
4. **Start capturing** — drop notes into `inbox/`, Jarvis sorts them.


## Philosophy

- **Plain markdown.** No proprietary format lock-in.
- **YAML frontmatter.** Structured metadata, searchable, scriptable.
- **PARA categories.** Four buckets, clear rules, nothing more.
- **Inbox-driven.** Capture first, organize second. Never interrupt flow.
- **Template-first.** Every note type has a template. Consistency without thinking.
